# 面向 B 站 / 小红书 / 抖音的本地视频工作流规划

## 目标

把当前项目从“本地生成口播视频”升级为“可持续生产发布物料”的工作台。核心输出不只是 `preview.mp4`，而是一套能直接服务哔哩哔哩、小红书、抖音的发布包：视频、字幕、封面、标题、简介、标签、风险检查和平台版本。

## 平台差异

### 哔哩哔哩

- 更适合中长视频、观点展开、系列内容和知识/情绪价值沉淀。
- 需要标题、封面、分区、标签、简介、原创声明、合集/系列归档。
- 重点能力：长视频结构、章节、封面、简介、标签、评论区引导。

### 小红书

- 更重封面、标题、收藏价值、个人经验感和低攻击性表达。
- 同一主题需要更短、更像笔记的表达方式。
- 重点能力：封面大字、短标题、多版本文案、图文摘要、风险词降噪。

### 抖音

- 更重前 3 秒钩子、竖版、强字幕、节奏和系列切片。
- 长口播需要拆成多条短视频，并保留连续观看动机。
- 重点能力：9:16 导出、短版切片、开头钩子评分、字幕字号和安全区。

## 功能框架

### 1. 内容策划层

- 选题 brief：受众、痛点、核心观点、争议点、平台适配方向。
- 脚本结构：开头钩子、问题展开、反转/解释、行动建议、结尾互动。
- 断言检查：性、心理、健康相关内容必须标注待核查。
- 多平台文案：同一主题生成 B 站长标题、小红书短标题、抖音钩子标题。

### 2. 生产层

- 文案导入：已支持 `.txt/.md/.docx` 和页面粘贴。
- 口播：当前使用 macOS 本地 TTS，后续可接本地 TTS/声音模型。
- 字幕：当前支持 SRT + ASS 硬字幕，后续需要平台安全区模板。
- 素材：已支持本地上传 BGM 和图片，也支持从中文免费 BGM 素材库粘贴音频 URL 导入。
- BGM 混音：已支持导入 BGM 后自动混入 MP4，并压低 BGM 音量保证口播优先。

### 3. 版本导出层

- B 站版：16:9 或 9:16，较完整结构，保留长论证。
- 小红书版：1:1 或 9:16，短观点、封面大字、强收藏点。
- 抖音版：9:16，短切片、强开场、字幕大、节奏快。
- 发布包：每个平台独立输出视频、标题、简介、标签、封面、检查单。

### 4. 审核与质量层

- 技术检查：分辨率、比例、时长、音量、字幕是否存在、文件大小。
- 内容检查：搬运风险、授权风险、医疗/心理/性暗示风险、夸大表达。
- 平台检查：标题长度、标签数、封面尺寸、竖版安全区。
- 回归验证：每次功能改动后运行 `make test`，并用本地页面做一次流程检查。

## 迭代清单

### 已完成

- 本地前端工作台。
- 文案导入和文件选择上传。
- 单一参考口播声音。
- SRT + ASS 高级硬字幕。
- MP4 预览和历史列表。
- 每期自动生成 `exports/publish-checklist.md`。
- B 站 / 小红书 / 抖音目标平台选择。
- 每个平台自动生成标题、标题变体、简介、标签、评论引导、转化 CTA。
- 每个平台自动生成 `metadata.json`、`publish.md`、封面 SVG/PNG。
- 按平台导出 16:9、4:5、9:16 视频。
- 抖音 / 小红书自动短视频切片。
- 平台发布包 zip 打包下载。
- 页面内展示平台评分、封面缩略图、发布检查、风险提示和优化建议。
- 页面内支持平台版本点击预览和发布文案复制。
- 生成历史按最近更新时间排序。
- 每个项目生成素材授权台账 `assets/licenses.md`。
- 每个项目生成发布复盘数据表 `exports/performance.csv`。
- BGM 支持两种入口：中文免费素材库导入、本地音频文件选择。
- 从素材库导入的 BGM 会保存来源 sidecar，并在授权台账里回填来源 URL。

### 下一轮优先级

1. 封面编辑器：支持在页面里调整封面标题、字号、位置、颜色和背景图。
2. 标题 A/B 测试库：保留不同标题点击假设、发布平台和效果回填。
3. 内容事实核查：把健康、心理、性相关断言拆成核查项和证据链接。
4. 平台发布日历：管理发布时间、发布状态、链接和复盘数据。
5. 数据复盘面板：手动录入播放、完播、点赞、收藏、评论、涨粉，反推选题和标题策略。
6. 素材授权编辑器：在页面直接补每张图、BGM、音效来源和授权备注。
7. 声音升级：接入更自然的本地 TTS 或已授权声音模型。
8. 素材检索：按脚本段落推荐本地素材库图片/视频片段。

## 当前项目判断

这个项目最适合先走“本地优先 + 发布包生成”的方向，而不是马上接真实平台 API 自动发布。原因是三个平台的审核、标题、封面和内容语境都需要人工判断；先把发布前物料和检查做扎实，比直接自动投稿更有用。
