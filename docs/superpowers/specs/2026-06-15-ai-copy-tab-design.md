# AI 文案 Tab 设计规格

## 目标

在链接素材工作台顶部新增独立的 `AI 文案` Tab。用户输入原始文案后，可选择本地 LM Studio、Gemini 网页版或 ChatGPT 网页版完成标题、正文、标签及平台文案改写。

本功能不把 ChatGPT Plus 或 Gemini 网页会员登录态伪装成 API。网页模型使用“生成提示词、复制、打开已登录网页”的辅助流程；LM Studio 使用本机 OpenAI 兼容接口直接生成结果。

## 页面位置

- `AI 文案` 位于顶部主导航第一项。
- 原有 `图文生产线`、`素材拆解`、`剪映视频`、`小红书图文`、`抖音图文`顺序后移。
- 页面沿用当前 Focus Studio 的白色工作区、青绿色强调色和紧凑生产工具风格。

## 页面结构

### 顶部任务栏

- 页面标题：`AI 文案`
- 状态提示：未开始、生成中、提示词已复制、生成完成、执行失败
- 主要操作：`开始生成`

### 左侧输入区

- 原始文案多行输入框
- AI 类型选择：
  - `本地 LM Studio`
  - `Gemini 网页版`
  - `ChatGPT 网页版`
- 任务类型：
  - 标题优化
  - 正文改写
  - 标签生成
  - 小红书改写
  - 抖音改写
- 改写强度：
  - 轻度：保留原结构，主要替换措辞
  - 标准：调整结构和表达节奏
  - 深度：重组结构，但不得改变事实和主旨
- 可选开关：
  - 保留或生成自然 emoji
  - 输出三个候选
- LM Studio 模式显示本地模型选择器。

### 右侧结果区

- 生成三个候选卡片
- 每个候选支持：
  - 复制
  - 设为当前结果
  - 回填到当前素材池项目
- 当前结果支持人工编辑
- 支持保存到本地 AI 文案历史
- 网页模式提供结果粘贴框，用于将 Gemini/ChatGPT 返回内容粘回工作台并拆分为候选。

## 执行模式

### LM Studio

1. 前端提交原文、任务类型、改写强度、emoji 设置、候选数量和模型。
2. 后端构建 OpenAI 兼容请求。
3. 后端使用封装的 `curl` 命令调用 `http://127.0.0.1:1234/v1/chat/completions`。
4. 请求体通过标准输入传给 `curl --data-binary @-`，避免命令行引号、换行和中文转义问题。
5. 后端解析 JSON 响应，规范化为三个候选并返回页面。
6. 超时、服务未启动、模型缺失和返回格式异常必须显示可操作的中文错误。

### Gemini 网页版

1. 根据当前设置生成完整提示词。
2. 将提示词写入系统剪贴板。
3. 新窗口打开 `https://gemini.google.com/app`。
4. 页面保持当前编辑状态，提示用户粘贴并发送。
5. 用户将网页结果粘回结果区，系统拆分候选。

### ChatGPT 网页版

流程与 Gemini 相同，新窗口打开 `https://chatgpt.com/`。

网页模式不自动读取网页回答、不抓取登录 Cookie、不模拟提交消息。这样避免登录状态失效、网页结构变化和账号风控。

## 提示词规则

所有任务共用以下约束：

- 保持原文事实、对象、立场和核心主旨。
- 通过结构、措辞、节奏、信息顺序和口语表达进行原创改写。
- 不编造商品参数、体验、功效、价格和促销信息。
- 避免绝对化、医疗功效和虚假承诺。
- 不使用“防判重”“规避审核”等表述要求模型欺骗平台；目标是提升原创表达质量。
- 输出纯 JSON 数组，默认三个候选。

任务差异：

- 标题：不含 `#`，控制平台长度，保持核心事件。
- 正文：保留信息完整度，增加自然段和真实口语感。
- 标签：每个标签以 `#` 开头，围绕原主题扩展。
- 小红书：适度 emoji、体验型表达、结构清晰，避免虚假种草。
- 抖音：标题更短、正文更直接、标签数量更少。

## 素材池联动

- 若用户从图文生产线选择了项目，再进入 AI 文案，页面可载入该项目标题、正文或标签。
- 选择候选后可回填：
  - 标题
  - 正文
  - 标签
- 回填通过现有 `/api/remix/pipeline/update` 接口保存。
- 未选择素材池项目时，回填按钮禁用，但复制与历史保存可用。

## 本地历史

保存到忽略 Git 的运行时文件：

`uploads/ai-copy-history.json`

每条记录包含：

- ID
- 创建时间
- AI 类型
- 模型
- 任务类型
- 改写强度
- 原文
- 提示词
- 候选
- 当前采用结果
- 关联素材池项目 ID

历史列表支持查看、恢复和删除。首版不提供云同步。

## 后端接口

### `GET /api/remix/ai-copy/history`

返回本地历史。

### `POST /api/remix/ai-copy/generate`

LM Studio 直接生成候选。

请求字段：

- `text`
- `task`
- `strength`
- `allow_emoji`
- `candidate_count`
- `model`

### `POST /api/remix/ai-copy/web-prompt`

生成网页模型提示词并返回目标 URL。前端负责复制提示词和打开页面。

请求字段：

- `provider`
- `text`
- `task`
- `strength`
- `allow_emoji`
- `candidate_count`

### `POST /api/remix/ai-copy/parse`

解析用户从 Gemini 或 ChatGPT 粘贴回来的文本，规范化为候选数组。

### `POST /api/remix/ai-copy/history/save`

保存或更新历史记录。

### `POST /api/remix/ai-copy/history/delete`

删除指定历史记录。

## 错误处理

- 原文为空：阻止执行并聚焦输入框。
- LM Studio 未运行：提示启动 `http://127.0.0.1:1234/v1` 服务。
- `curl` 不存在：回退到现有 Python HTTP 调用，并在日志标记执行方式。
- 请求超过 180 秒：停止进度状态并允许重试。
- 网页被浏览器拦截：保留提示词并显示“再次打开”和“复制提示词”。
- 网页返回内容无法解析：保留原始粘贴内容，允许作为单个候选使用。

## 验收标准

1. 顶部第一项显示 `AI 文案`，不破坏现有 Tab。
2. LM Studio 能在页面内生成并展示候选。
3. LM Studio 请求由安全封装的 `curl` 执行，中文和换行文本不丢失。
4. Gemini 和 ChatGPT 能复制完整提示词并打开正确网页。
5. 网页返回内容可粘回并拆分成候选。
6. 候选可复制、编辑、保存历史并回填素材池。
7. 生成中按钮不可重复点击，展示动态进度。
8. 桌面与移动布局无横向溢出。
9. 后端单元测试、前端语法检查和浏览器关键流程回归通过。

## 非目标

- 不自动读取 ChatGPT/Gemini 网页答案。
- 不抓取或保存网页 Cookie、Token 或登录凭证。
- 不承诺网页会员权益等同于 API 调用额度。
- 不在首版接入 OpenAI API Key 或 Gemini API Key。
